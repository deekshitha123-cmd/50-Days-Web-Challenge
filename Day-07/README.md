# 📝 Day 07 – Form UI & Interactive States (Membership Application)

## 📌 Project Overview

This project is part of my **50 Days Web Development Challenge**. The objective of Day 07 was to design and develop a responsive **Membership Application Form** using **HTML5** and **CSS3**. The focus was on creating an accessible form with interactive UI states that provide clear visual feedback during user interactions.

## 🚀 Features

- Responsive Membership Application Form
- Accessible form structure using `label` and `for` attributes
- Custom-styled input fields and textarea
- Interactive `:focus` state for active inputs
- Hover effect on the submit button
- Disabled button styling using `:disabled`
- HTML5 built-in form validation
- Real-time validation using `:valid` and `:invalid`
- Clean and modern user interface
- Mobile-friendly responsive design

## 🎯 CSS Pseudo-classes Used

| Pseudo-class | Purpose |
|--------------|---------|
| `:focus` | Highlights the active input field |
| `:hover` | Changes button appearance on hover |
| `:disabled` | Styles disabled buttons |
| `:valid` | Indicates valid user input |
| `:invalid` | Highlights invalid input |
| `:placeholder-shown` | Prevents validation styles before typing |

## ♿ Accessibility Features

- Proper linkage between `label` and `input` using matching `for` and `id` attributes.
- Clickable labels for improved usability.
- Visible focus indicators for keyboard users.
